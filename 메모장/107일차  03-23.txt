이미지 업로드 기능 구현: 클라이언트가 이미지를 업로드할 수 있는 기능을 개발합니다. 이를 위해서는 HTML 폼을 생성하고, 해당 폼을 통해 이미지를 서버에 전송하는데 필요한 컨트롤러와 서비스를 구현해야 합니다.

오버로딩은 같은 이름의 메서드를 다양한 매개변수로 정의하는 것이고, 오버라이딩은 상위 클래스의 메서드를 하위 클래스에서 다시 정의하는 것입니다.

List(리스트):
리스트는 순서가 있는 요소의 집합으로, 중복된 요소를 허용합니다.
주요 구현 클래스로는 ArrayList, LinkedList, Vector가 있습니다.

Set(집합):
집합은 순서가 없는 요소의 집합으로, 중복된 요소를 허용하지 않습니다.
주요 구현 클래스로는 HashSet, LinkedHashSet, TreeSet가 있습니다.

Map(맵):
맵은 키-값 쌍의 집합으로, 키는 중복될 수 없으며, 각 키는 하나의 값에 매핑됩니다.
주요 구현 클래스로는 HashMap, LinkedHashMap, TreeMap이 있습니다.

final 변수:
final 변수는 상수를 나타냅니다. 즉, 한 번 초기화되면 변경할 수 없습니다.
final 변수는 선언 시점에 직접 초기화하거나 생성자에서 초기화할 수 있습니다.
다음과 같이 선언됩니다: final int MAX_VALUE = 100;
메서드 내부에서는 값의 변경이 허용되지 않으며, 따라서 값을 다시 할당할 수 없습니다.

final 메소드:
final 메서드는 하위 클래스에서 재정의(오버라이딩)할 수 없습니다.
즉, final로 선언된 메서드는 상속되는 클래스에서 변경될 수 없습니다.
final 메서드는 주로 부모 클래스에서 특정 동작이 변경되지 않아야 하는 경우에 사용됩니다.
다음과 같이 선언됩니다: final void methodName() { ... }

final 클래스:
final 클래스는 하위 클래스를 가질 수 없습니다. 즉, 상속될 수 없는 클래스입니다.
final로 선언된 클래스는 더 이상 확장될 수 없으므로, 다른 클래스에서 상속하여 새로운 하위 클래스를 만들 수 없습니다.
final 클래스는 보안적인 이유나 클래스의 불변성을 보장하기 위해 사용될 수 있습니다.
예를 들어, java.lang.String 클래스는 final로 선언되어 있어서 상속할 수 없습니다.

Apache Maven:
Apache Maven은 Java 기반 프로젝트의 빌드, 의존성 관리 및 문서화를 자동화하는 도구입니다. XML 기반의 프로젝트 설정 파일(pom.xml)을 사용하여 프로젝트를 관리합니다.

빌드 관리 도구는 프로젝트의 규모와 요구 사항, 개발자의 선호도에 따라 선택될 수 있습니다. 이러한 도구들은 프로젝트의 빌드 과정을 효율적으로 자동화하여 개발자의 생산성을 향상시키고, 프로젝트의 관리를 간편하게 해줍니다.

전송 방식 : 
GET: 데이터를 서버로 전송하기 위해 URL의 쿼리 문자열에 데이터를 포함하여 전송됩니다. 이는 주로 데이터의 검색이나 조회에 사용됩니다. 브라우저에서 웹 페이지를 요청할 때 사용되는 메서드입니다.
POST: 데이터를 HTTP 요청 본문에 담아 서버로 전송됩니다. 이는 주로 데이터의 전송이나 수정에 사용됩니다. 폼 데이터를 전송하거나 파일을 업로드할 때 사용됩니다.

데이터 전달 방식:
GET: 데이터가 URL의 쿼리 문자열에 노출되므로 보안에 취약합니다. 또한 전송할 수 있는 데이터의 길이에 제한이 있습니다.
POST: 데이터가 HTTP 요청 본문에 담겨 전송되기 때문에 URL에 노출되지 않습니다. 또한 전송할 수 있는 데이터의 크기에 제한이 없습니다.

기본키와 외래키의 차이점 :
기본키는 테이블 내에서 각 행을 고유하게 식별하는 데 사용되는 열이고, 외래키는 다른 테이블의 기본키와 관련하여 테이블 간의 관계를 설정하는 데 사용됩니다. 함께 사용되어 데이터베이스에서 데이터의 일관성과 정확성을 보장하는 데 중요한 역할을 합니다.
